1 The Luxembourg Income Study

1.1 Purpose and Content

1.1.1 Mission

  • enable cross-national comparative research on socio-economic outcomes and underlying institutional factors (e.g. tax and transfer system)
  • academic guidance by prominent economics and other social scientist scholars
  • director: Prof. Peter Lanjouw (VU)

1.1.2 Coverage

  • micro data on income, wealth, and consumption
  • on household or individual level
  • harmonized data definitions across time and space

1.2 Data Overview

1.2.1 Website

Source: LIS Data Center

1.2.2 Sources

  • existing country-specific micro data surveys and (samples of) administrative tax records
  • e.g., EU-SILC, ECHP; FES/FRS; GSOEP; CPS; SCF
  • harmonization according to common template
  • waves: annualization
  • sampling weights

1.2.3 All Countries

  • 3 data bases: LIS, LWS, ERF-LIS
  • 1063 cross sectional data sets from 58 countries (Table 1)
  • varying sample sizes (mostly, several 000)
  • covering 21/27 EU countries and 33/38 OECD countries

1.2.4 LIS

  • 953 data sets from 53 countries (Figure 2)
  • average # years per country: 17.5 (but often gaps)
  • some countries’ data span 5 or more decades
    • US: 61 years (1963-2023)
    • UK: 48 years (1968-2021)
    • CA: 45 years (1971-2021)
  • longest consecutive time series of cross sections
    • US: 61 years (1963-2023)
    • UK: 46 years (1976-2021)
    • DE: 38 years (1983-2020)
  • household and individual income files
  • often before and after taxes and transfers
  • core demographics, education, labor and health

1.2.5 LWS

  • 115 data sets from 21 countries
  • cross sections typically every 3-5 years

1.2.6 ERF-LIS

  • 29 data sets from 7 countries (typically middle-east and Africa)
  • often singletons

1.4 Data Analysis

1.4.1 Data Scope

1.4.1.1 Micro Data

  • LIS data sets are named nnyyt (e.g. “nl12h”) where

    • nn is ISO2 code of country, e.g., “nl” for Netherlands (Table 1)
    • yy is year, e.g., “12” for 2012
    • t is type, e.g. “h” for household level (or “i” for individual)

  • LWS data sets are named nnyywt where

    • ‘w’ is a fixed name component to differentiate it as LWS file from LIS files
    • t has values “h” (household), “p” (person), “r” (implicate; for multiple imputations)

  • data sets are \(N\times K\) matrices with \(N\) records (one per row) and \(K\) variables (one per column)

  • analysis on household or individual level

  • can append or combine data sets per country across years, or across countries, and combine household level data with invdividual level data

  • no panel data, cannot follow individuals/households over time

  • pseudo-panels: group-level analysis (follow birth cohorts, education groups, geographical subunits, etc. over time)

1.4.1.2 Variables

  • demographics (household relationships, family composition, marital status, number of children, sex, education, immigration status/country of origin, etc.)
  • labor market variables (participation, labor supply, leave schemes, experience, occupation, industry and wages)
  • health related variables (disability indication and health status)
  • income concepts: household level / individual level
  • income concepts: market or primary income / disposable net income / income by source (work, social benefits, pensions)
  • LIS: aggregate consumption categories; ERFLIS: detailed consumption expenditure
  • documentation
    • LIS data overview
    • detail: Section 1.3.3
  • Note:
    • not every country has all variables for every year (or variables may exist but have no content)
    • some variables have country-specific content (variable names typically suffixed by _c)

1.4.1.3 Macro Data

  • LIS data servers also host additional data at an aggregate/macro level that can be directly linked to micro data
    • CPI and PPP deflators
    • Policy parameters

1.4.2 Data Analysis

1.4.2.1 Analysis Types

  • descriptive statistics, including graphical
  • regression or other type of statistical modeling work (estimation and inference)

1.4.2.2 Remote Execution

  • write your code (saved on your device or drive)
  • copy-paste it into the LISSY mask and submit the ‘job’
  • job will be added to internal job-queue and executed when time is found to do so
  • jobs have a unique job ID
  • LISSY server sends back output
    • plain text (ASCII) job log, e.g.
      • summary tables
      • regression tables
    • image files for graphs (e.g., .png)

1.4.2.3 Hints and Warnings

  • MUST NOT (try) to look at data records, e.g., NEVER view/list/print individual records or values
  • DO NOT resubmit a scheduled job before completion, wait until you receive output for your job ID
  • BREAK UP long programs into smaller code chunks, test them thoroughly
  • ECONOMIZE on number of data sets used simultaneously and/or number of variables and/or number of observations used
  • see LISSY Coding Best Practices and Managing LISSY Jobs and Listings – FAQs
  • for international comparison, LIS recommends Top and Bottom Coding

2 LISSY

2.1 Session Workflow

  • have your code ready on file (to be copy/pasted from your own device)

    • an alternative is to code directly into the UI code editor; since it’s quite simple, few users probably choose to do so

  • open LISSY UI

  • select in the mask:

    Project: LIS/LWS, etc.

    Package: R/Stata/SPSS, etc.

    Subject: Free text of choice

  • copy/paste code into code editor (‘edit job’ tab)

  • press the green ‘play’ button (upper left corner of editor) that appears after mask is completed

  • job particulars, including output will be shown in “recent jobs” tab

  • tabs:

    • job text: your submitted code
    • listing: plain text results and error messages
    • graphs (if any) for download

  • you also will receive an email with the ‘job listing’

2.2.1.3.3 Working with household income variables: top and bottom coding and equivalence scales
wmean <- function(x, weight= NULL) {
    if (is.null(weight))
    weight <- rep(1, length(x))
    y <- x[which(!is.na(x))]
    wgt <- weight[which(!is.na(x))]
    wmean <- sum(y*wgt/sum(wgt))
    return(wmean)
}

# weighted percentile (e.g. median)
wNtile <- function(var, wgt = NULL, split) {
    if (is.null(wgt))
    wgt <- rep(1, length(var))
    x <- var[order(var)]
    y <- wgt[order(var)]
    z <- cumsum(y) / sum(y)
    cop <- rep(NA,length(split))
    for (i in 1:length(cop)) {
        cop[i] <- x[Find(function(h) z[h] > split[i], seq_along(z))]
    }
    return(cop)
}

# apply top and bottom coding
topBottom <- function(var, botline, topline) {
    tb <- ifelse(var < botline, botline, var)
    tb[tb > topline] <- topline
    return(tb)
}

setups <- function(data_file) {
    vars <- c('dhi', 'hifactor', 'hpub_i', 'hpub_u', 'hpub_a', 'hiprivate', 'hxitsc', 'hpopwgt', 'nhhmem', 'grossnet')

    botline <- 0

2.2.1.3.4 Inequality: The Gini Index
# Gini
gini <- function(x,weight) {
    ox <- order(x)
    x <- x[ox]
    weight <- weight[ox]/sum(weight)
    p <- cumsum(weight)
    nu <- cumsum(weight*x)
    n <- length(nu)
    nu <- nu/nu[n]
    res <- sum(nu[-1]*p[-n])-sum(nu[-n]*p[-1])
    return(res)
}

2.2.1.3.8 Producing compact and concise output
gini <- function(x,weight) {
    ox <- order(x)
    x <- x[ox]
    weight <- weight[ox]/sum(weight)
    p <- cumsum(weight)
    nu <- cumsum(weight*x)
    n <- length(nu)
    nu <- nu/nu[n]
    res <- sum(nu[-1]*p[-n])-sum(nu[-n]*p[-1])
    return(res)
}

wNtile <- function(var, wgt, split) {
    x <- var[order(var)]
    y <- wgt[order(var)]
    z <- cumsum(y) / sum(y)
    cop <- rep(NA,length(split))
    for (i in 1:length(cop)) {
        cop[i] <- x[Find(function(h) z[h] > split[i], seq_along(z))]
    }
    return(cop)
}
